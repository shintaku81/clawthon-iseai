Operator Guide

Clawthon ISEAI 運営マニュアル

このマニュアルはハッカソン運営者向けです。VM作成・コスト管理・トラブルシューティングまでを網羅しています。

📖

システム概要

Clawthon ISEAIは、ハッカソン参加者一人ひとりに専用の開発環境（VM）を提供するプラットフォームです。運営者はこの管理コンソールから全VMを一元管理できます。

コンポーネント役割サーバー
管理コンソール 運営者向けダッシュボード（このアプリ） polaris (管理VM)
参加者VM 各参加者の開発環境（OpenClaw + VSCode + OpenHands） vega, altair, ... (参加者VM)
OpenClaw (Claude Code CLI) メイン — ターミナルでAIとコーディング 各参加者VM (VSCode ターミナル)
OpenHands サブ — ブラウザUIのAIエージェント（任意） 各参加者VM :3000
code-server ブラウザ版VSCode 各参加者VM :8080
nginx + SSL リバースプロキシ、Let's Encrypt証明書 各VM (Let's Encrypt)
📋

事前準備（前日まで）

  1. 1
    参加者リストの確定
    氏名・メールアドレスをスプレッドシートで整理する。参加者数分のAPIキー（AnthropicまたはOpenRouter）を準備する。
  2. 2
    GCPプロジェクトの確認
    clawthon-iseai プロジェクトの請求が有効であること、Compute Engine APIが有効であることを確認する。
  3. 3
    管理コンソールへのログイン確認
    https://console.iseai.neuratools.ai にアクセスし、メールアドレス + パスワードでログインできることを確認する。
  4. 4
    テスト用VMの作成（推奨）
    前日に1台テストVMを作成して、OpenHands・VSCode・SSLが正常に動くか確認する。問題なければ削除する。
💡 APIキーはAnthropicの場合 sk-ant-...、OpenRouterの場合 sk-or-v1-... の形式です。LiteLLM Proxyを使う場合はProxyのURLとキーを設定してください。
🚀

ハッカソン当日の開始手順

  1. 1
    参加者をダッシュボードに追加
    ダッシュボード下部の「参加者を追加」フォームから名前・メールを登録する。
  2. 2
    APIキーを各参加者に設定
    各参加者行の「▼ 編集」を開き、APIキーを貼り付けて「保存」する。
    重要 APIキーを設定してから「作成」ボタンを押すこと。
  3. 3
    VMを作成
    参加者行の「作成」ボタンを押す。バックグラウンドで5〜10分かかる。ステータスが RUNNING になれば完了。
  4. 4
    URLを参加者に共有
    VMが起動したら https://pXX.iseai.neuratools.ai を参加者に伝える。OpenHandsとVSCodeの両方が使える。
⚠️ VM作成からOpenHandsが使えるようになるまで約5〜10分かかります。参加者に「少し待つよう」伝えてください。
🏁

ハッカソン終了・後片付け

  1. 1
    参加者の成果物をバックアップ
    各VMの /opt/clawthon/workspace に成果物がある。必要であればSFTPまたはGCSにバックアップする。
  2. 2
    VMを停止または削除
    コスト節約のため、不要なVMは「停止」または「削除」する。停止はデータを保持、削除はデータも消える。
  3. 3
    コスト確認
    GCPコンソールで請求額を確認する。ダッシュボードの概算コスト表示も参考にすること。
🖥️

参加者VM 管理

操作説明所要時間データ
作成 新規VMを作成。Docker・OpenHands・code-server・nginxを自動セットアップ 5〜10分 新規
起動 停止中のVMを再起動。Dockerコンテナも自動起動 1〜2分 保持
停止 VMをシャットダウン。ディスクデータは保持される 30秒 保持
削除 VMとDNSレコードを完全削除。データも消える 1分 消去
⚠️ SPOT（プリエンプティブル）インスタンスはGCPのリソース不足時に自動停止されることがあります。停止した場合は「起動」ボタンで再起動してください。
🦀

OpenClaw / Claude Code CLI

OpenClaw は Claude Code CLI のハッカソン名称です。参加者は VSCode のターミナルから claude コマンドを使って、AI と対話しながらコーディングできます。

参加者の使い方

  1. VSCode (/code/) を開く
  2. ターミナルを起動 (Ctrl+`)
  3. claude と入力して Enter
  4. APIキーを入力（初回のみ）
  5. AIと会話しながらコーディング開始

バージョン確認

claude --version

インストール場所

cloud-init により VM 作成時に自動インストール済み (/usr/local/bin/claude)

ℹ️ APIキーは運営から参加者に配布します。参加者が初めて claude を実行したとき、APIキーの入力を求められます。
🤖

OpenHands（サブ機能）

OpenHandsは任意のツールです。ブラウザUIでAIエージェントを操作したい場合に使います。メインツールはOpenClaw (Claude Code CLI)です。

  • 参加者URL: https://pXX.iseai.neuratools.ai/openhands/
  • 管理コンソールからチャット: ダッシュボードの「チャット」ボタンで管理画面内に埋め込み表示
  • バージョン変更: 「OpenHands 設定」パネルでバージョンを選択 → 「保存 & 再起動」
  • ポート: 各VM上の :3000 で動作（nginxが /openhands/ にプロキシ）

バージョン管理

バージョンステータス推奨
latest 最新リリース（自動更新） 推奨
0.29 安定版 —
0.28 旧バージョン —
💡 OpenHandsが「未起動」と表示されている場合は、APIキーが未設定か、コンテナの起動に時間がかかっています。VM作成から5分ほどお待ちください。

VSCode (code-server) の操作

参加者はブラウザ上でVSCodeを使ってコードを編集・デバッグできます。ターミナルも使用可能です。

  • 参加者URL: https://pXX.iseai.neuratools.ai/code/
  • パスワード: clawthon2026
  • ワークスペース: /opt/clawthon/workspace にマウント済み
  • ポート: 各VM上の :8080 で動作
ℹ️ code-serverのパスワードは全参加者共通です。セキュリティが必要な場合は個別に変更してください（VM内 /home/ubuntu/.config/code-server/config.yaml）。
💴

コスト管理

ダッシュボードのヘッダーに概算コストをリアルタイム表示しています。

リソーススペック単価（概算）
管理VM (polaris) e2-small, 常時稼働 $0.017/時 ≈ ¥2.5/時
参加者VM (SPOT) e2-standard-2, プリエンプティブル $0.025/時 ≈ ¥3.75/時/台
Cloud DNS ゾーン1個 $0.20/月
静的IP 使用中は無料、未使用は $0.010/時 運用中は無料

コスト試算例：参加者10人・2日間（48時間）

# 参加者VM（SPOT） $0.025 × 48時間 × 10台 = $12.00 # 管理VM（常時稼働） $0.017 × 48時間 = $0.82 # 合計（150円/ドル換算） $12.82 ≈ ¥1,923
💡 ハッカソン終了後は必ずVMを「停止」または「削除」してください。SPOTインスタンスは通常の約70%引きですが、起動したままだとコストが積み上がります。

サーバー監視

ダッシュボードの「監視」ボタンで各VMのリアルタイムメトリクスを確認できます。

指標確認方法目安
CPU使用率 監視パネルに % 表示 OpenHands実行中は 70-90% が正常
メモリ使用率 監視パネルに % 表示 80% 超えたら要注意
ディスク使用率 監視パネルに % 表示 90% 超えたらワークスペースを整理
OpenHandsバージョン 監視パネルに表示 設定バージョンと一致していること
コンテナ状態 openhands: running / code-server: running 両方 running が正常
📄

ログ確認

ダッシュボードの「ログ」ボタンから各VMのログを確認できます。

  • OpenHands ログ: Dockerコンテナのstdout。エラー・進捗確認に使用
  • 管理コンソール ログ: systemdサービスのログ。API呼び出し・エラー追跡に使用

SSHで直接確認する場合：

# OpenHandsのリアルタイムログ sudo docker logs -f openhands --tail 50 # 管理コンソールのサービスログ sudo journalctl -u clawthon-console -f --lines 50 # nginxのエラーログ sudo tail -f /var/log/nginx/error.log

トラブルシューティング

症状原因対処
OpenClaw が動かない claude コマンドが見つからない npm install -g @anthropic-ai/claude-code
VMが「TERMINATED」 SPOTインスタンスがGCPに回収された 「起動」ボタンで再起動
OpenHandsが「未起動」のまま APIキー未設定 / コンテナ起動中 APIキーを確認。VM作成後5分待つ
SSL エラー (ERR_CERT_*) 証明書の更新失敗 / nginx停止 sudo certbot renew --nginx を実行
管理コンソールにアクセスできない サービス停止 / デプロイ失敗 sudo systemctl restart clawthon-console
VM作成が失敗する GCPクォータ超過 / 認証エラー GCPコンソールでエラーを確認。クォータ申請が必要な場合あり
OpenHandsが遅い / 固まる メモリ不足 / API制限 コンテナ再起動: sudo docker restart openhands
VSCodeにアクセスできない code-serverコンテナが停止 sudo docker restart code-server

SSHコマンド集

緊急時やコンソールで操作できない場合は、直接VMにSSHして対応します。

# ── 管理VMへSSH ────────────────────────────── gcloud compute ssh clawthon-management \ --project=clawthon-iseai \ --zone=asia-northeast1-b # ── 参加者VMへSSH ──────────────────────────── gcloud compute ssh clawthon-p01 \ --project=clawthon-iseai \ --zone=asia-northeast1-b # ── コンソール再起動 ────────────────────────── sudo systemctl restart clawthon-console sudo systemctl status clawthon-console # ── Dockerコンテナ再起動 ────────────────────── sudo docker restart openhands sudo docker restart code-server sudo docker ps # ── SSL証明書の更新 ─────────────────────────── sudo certbot renew --nginx sudo nginx -t && sudo systemctl reload nginx # ── nginxのリロード ─────────────────────────── sudo nginx -t sudo systemctl reload nginx # ── ディスク確認 ────────────────────────────── df -h du -sh /opt/clawthon/workspace/*
