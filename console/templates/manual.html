運営者向け操作ガイド

📋当日の準備手順

ハッカソン開始前に以下の順で準備してください。

  1. 参加者情報を登録 — ダッシュボード下部のフォームから名前・メールを追加
  2. APIキーを設定 — 各参加者行の「▼ 編集」を開いてAnthropicまたはOpenRouterのAPIキーを貼り付け → 保存
  3. VMを作成 — 「作成」ボタンを押す（バックグラウンドで5〜10分かかります）
  4. 参加者にURLを共有 — https://p01.iseai.neuratools.ai の形式でURLを配布
💡 APIキーを設定してから「作成」ボタンを押してください。APIキーはVMの起動時に自動で注入されます。
🖥️参加者VM 管理
操作説明所要時間
作成 新規VMを作成。cloud-initでDocker・OpenHands・code-serverを自動セットアップ 5〜10分
起動 停止中のVMを再起動（SPOTインスタンスは自動停止する場合あり） 1〜2分
停止 VMをシャットダウン。データは保持される 30秒
削除 VMとDNSレコードを完全削除。データも消えるので注意 1分
⚠️ SPOT（プリエンプティブル）インスタンスはGCPのリソース不足時に自動停止されます。停止した場合は「起動」ボタンで再起動してください。
🤖OpenHands の操作

OpenHandsはAIエージェントが自律的にコードを書くツールです。参加者は自然言語でタスクを指示できます。

  • 参加者URL: https://p01.iseai.neuratools.ai/openhands/
  • コンソールからチャット: ダッシュボードの「チャット」ボタンで管理画面内に埋め込み表示
  • バージョン変更: 「OpenHands 設定」パネルでバージョンを選択 → 「設定を保存 & VM再起動」
💡 OpenHandsが「未起動」と表示されている場合は、APIキーが未設定か、コンテナの起動に時間がかかっています。VM作成から5分ほどお待ちください。
💻VSCode (code-server) の操作

参加者はブラウザ上でVSCodeを使ってコードを編集できます。

  • 参加者URL: https://p01.iseai.neuratools.ai/code/
  • パスワード: clawthon2026
  • ワークスペース: /opt/clawthon/workspace にマウント済み
💴コスト管理

ダッシュボードのヘッダーに概算コストを表示しています。

リソーススペック料金（概算）
管理VM (polaris) e2-small, 常時稼働 $0.017/時 ≈ ¥2.5/時
参加者VM (vega等) e2-standard-2, SPOT $0.025/時 ≈ ¥3.75/時/台
Cloud DNS ゾーン1個 $0.20/月

例：参加者10人、2日間（48時間）のコスト概算：

管理VM: $0.017 × 48 = $0.82 参加者VM: $0.025 × 48 × 10台 = $12.00 合計: $12.82 ≈ ¥1,923（150円/ドル換算）
💡 ハッカソン終了後は「停止」または「削除」でコストを抑えてください。SPOTインスタンスは通常の約70%引きです。
🔧トラブルシューティング
症状原因対処
VMが「TERMINATED」になった SPOTインスタンスがGCPに回収された 「起動」ボタンで再起動
OpenHandsが「未起動」のまま APIキー未設定 / コンテナ起動中 APIキーを確認。VM作成後5分待つ
ページが開かない（SSL エラー） 証明書の更新失敗 / nginx停止 VM再起動後にcertbot renewを実行
管理コンソールにアクセスできない サービス停止 管理VMにSSHして sudo systemctl restart clawthon-console

緊急SSHコマンド:

# 管理VMへSSH gcloud compute ssh clawthon-management --project=clawthon-iseai --zone=asia-northeast1-b # 参加者VMへSSH gcloud compute ssh clawthon-p01 --project=clawthon-iseai --zone=asia-northeast1-b # コンソール再起動 sudo systemctl restart clawthon-console sudo systemctl status clawthon-console # OpenHands再起動 sudo docker restart openhands # SSL証明書更新 sudo certbot renew --nginx
